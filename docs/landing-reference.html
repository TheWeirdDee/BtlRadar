Architecture

Three agents.
One gateway.

BTL Runtime routes each agent to the cheapest model capable of its job. Agent 1 handles thousands of transactions cheaply. Agent 3 fires rarely — only when it actually matters.

AGENT 01
High-Speed Screener
Llama 3 · via BTL Runtime
Processes 100% of incoming transactions in real time. Screens for volume spikes, dev wallet movement, gas anomalies, and liquidity shifts. Runs continuously at near-zero cost.
Triggers: always running · flags suspicious txs
AGENT 02
Forensic Analyst
GPT-4o mini · via BTL Runtime
Activates only when Agent 1 flags something. Inspects wallet history, deployer identity, liquidity patterns, and cross-references against known rug signatures.
Triggers: on Agent 1 flag · escalates if high risk
AGENT 03
Tactical Judge
Claude Sonnet · via BTL Runtime
Fires only when Agent 2 confirms high risk. Produces a plain-English verdict with rug probability score. Triggers alerts across web, Telegram, and X simultaneously.
Triggers: on Agent 2 escalation · fires alerts
